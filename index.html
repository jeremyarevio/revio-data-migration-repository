Alarm 7 SQL

📤 Report Export

Alarm 7 SQL is a billing and service CRM built for security alarm dealers, backed by a SQL database with open ODBC connectivity. Data can be exported via the built-in reports engine (recommended for most users) or directly from the database for IT teams needing raw extracts.

Option A: Built-In Reports
Option B: Direct DB Access

No technical knowledge required. Most migration data can be exported directly from the Reports menu.

1
Log in as an Admin user

Use an account with full access to the Reports section and all customer data. Limited users may not see all report options.

2
Navigate to the Reports section

From the Main Menu, select Reports. Reports are organized by category — look for Customer, Billing/Recurring, Service, and Inventory groups.

3
Select the report you need and set filters

For migration purposes, export all active records with no date filter applied unless otherwise directed by your Rev.io Sales Representative.

4
Export to Excel or CSV

Once a report is generated, look for an Export or Print option in the toolbar. Choose Excel (.xls) or CSV for easiest compatibility with Rev.io import templates.

5
Use Custom Reports for missing fields

Go to Reports → Custom Reports to select specific fields from the database if a built-in report doesn't include what you need.

6
Repeat for each data category

Work through the categories in the table below and save all exports to a single folder labeled with your company name and export date.

💡
Alarm 7 SQL supports Outlook integration. If customer email addresses are incomplete in the main reports, try File → Export to Outlook as a supplemental source.

ℹ️
This option requires someone familiar with your Windows server environment and basic SQL. Z-Micro explicitly states they never prevent you from accessing your own data — the database is open and ODBC-accessible.
1
Identify your database server

Alarm 7 SQL is typically installed as a shared network application. Check with the person who manages your installation for the server name or IP address.

2
Locate the data directory

The default data path is typically C:\ALARMWIN\ or the network share path configured during setup. Verify by checking the Alarm 7 SQL workstation shortcut or network share configuration.

3
Connect via ODBC

Open Windows Control Panel → Administrative Tools → ODBC Data Sources. Create a new System DSN pointing to the Alarm 7 SQL database. Then use any SQL-capable tool (SQL Server Management Studio, Excel Power Query, Microsoft Access, etc.) to query the tables directly.

4
Export tables to CSV

Query the relevant tables and export to CSV. Key tables include the customer master, recurring charges, invoice history, and service tickets. Your Rev.io Sales Representative can advise on which tables map to which import templates.

⚠️
If your Alarm 7 SQL is hosted or managed by a third party, you may need to request a database backup or extract directly from them. Contact Z-Micro Technologies at (877) 864-5010 if you need help obtaining your data.
What to Send Rev.io
  • ☐ Customer / Account List — all active accounts with name, address, phone, email, and account number
  • ☐ Recurring Charges (RMR) by Account — billing line items, amounts, frequencies, and service descriptions
  • ☐ Open Invoices / A/R Aging — outstanding balances that need to carry over into Rev.io
  • ☐ Service / Product Catalog — services and parts used in recurring billing and invoices
  • ☐ Equipment per Site (if applicable) — installed equipment records per customer
💡
Use one file per data category with a header row. Avoid merged cells or summary rows — raw report exports from Alarm 7 SQL are ideal.

AlarmBiller

📤 Report Export

AlarmBiller provides built-in export tools accessible directly from list views and the Reports section. No API setup is required. Follow the methods below to pull all data needed for your Rev.io migration.

⚠️
Before you start: Confirm your AlarmBiller account has Administrator or Manager role permissions. Some reports and export functions are role-restricted. If you can't access something, contact your AlarmBiller admin or open a ticket at support.boldgroup.com.
Method 1 — Export from Grid / List Views

Most list views in AlarmBiller (Customers, Invoices, RMR, Payments) allow direct export to a spreadsheet.

1
Navigate to the section you want to export

Go to Customers, Invoices, RMR, or Payments — whichever data you need.

2
Apply filters (optional)

Set any date range, status, or customer type filters to scope your dataset before exporting.

3
Locate the Export / Download icon

Look in the upper-right area of the grid for a spreadsheet or CSV icon. If you don't see it, check the column settings or options menu — it may be nested there.

4
Download the file

Click the icon to download the visible data as a CSV or Excel file. Repeat for each data section you need.

Method 2 — Reports Section

AlarmBiller's Reports section provides structured financial and operational reports with download options.

1
Click Reports in the top navigation
2
Select the appropriate report

Key reports for migration:

  • RMR Report — recurring billing schedules per customer
  • Aging Report — open invoices by age
  • Customer List — active and inactive customers
  • Invoice Report — invoice history by date range
  • Payment Report — payment history
3
Set your date range and filters, then click Run / Generate
4
Download the report

Look for a Download, Export, or Print option. Most reports support PDF and/or CSV output.

5
Label and save each file

Save to a clearly labeled folder, e.g. AlarmBiller_Export_YYYYMMDD, and send to your Rev.io implementation team.

Method 3 — QuickBooks IIF Export (Financial History)

If you need a complete record of invoices, credits, and payments, AlarmBiller can export a QuickBooks-compatible IIF file.

1
Go to Setup → QuickBooks

Review the QuickBooks documentation on that page before proceeding.

2
Click the Export tab

Enter your desired export date. AlarmBiller will export all reconciled transactions up to that date.

3
Click Export (upper-right) to queue the file
4
Download the IIF file

Click the Download button next to the export entry once it has generated.

ℹ️
Only reconciled transactions are included in the IIF export. Customer records and RMR data are not available through this path — use Methods 1 and 2 for those.

ConnectWise Manage

🔑 API Access

This guide walks you through creating an API key in ConnectWise Manage so Rev.io can securely pull your data for migration. The process takes about 10 minutes and does not require any software installation.

What We Need From You
ItemDescriptionExample
Site URLThe domain of your ConnectWise instancena.myconnectwise.net
Company IDThe company name used on the login screenyourcompanyname
Public KeyGenerated in Step 4aBcDeFgHiJkLmNoP
Private KeyGenerated in Step 4 — shown once onlyaBcDeFgHiJkLmNoP
🔒
These credentials provide read-only access. Rev.io will not use them to make any changes to your system.
Setup Steps
1
Find Your Site URL and Company ID

Log into ConnectWise Manage. Look at the address bar — your URL will look like:
https://na.myconnectwise.net/v4_6_release/ConnectWise.aspx

  • Site URL: the domain portion, e.g. na.myconnectwise.net
  • Company ID: the name you type in the Company field on the login screen
2
Create an API Member Account

An API Member is a special non-login account for API access. It does not consume a regular user license.

  • Go to System → Members and click + to create a new member.
  • Set Member ID to APIIntegration (or any name you prefer).
  • Set License Class to API (not Full or Non-billable).
  • Assign the security role created in Step 3, then click Save.
3
Create a Security Role with Read Permissions

Create a role called API Read Only under System → Security Roles and enable Inquire (read) access for the following:

CategorySections
CompaniesCompanies, Contacts, Configurations, Sites
FinanceAgreements, Invoices, Billing Terms, Tax Codes
ProcurementProducts, Product Catalog
Service DeskTickets, Boards, Priorities
SalesOpportunities, Activities
Time & ExpenseTime Entries, Expense Entries
ScheduleSchedule Entries
SystemMembers, Locations

After saving the role, go back to the API Member and assign it.

4
Generate API Keys
  • While viewing the API Member record, click the API Keys tab.
  • Click + to generate a new key pair; give it a description such as Rev.io Data Migration.
  • Click Save.
⚠️
ConnectWise will display the Public Key and Private Key. Copy both immediately — the Private Key is only shown once and cannot be retrieved later.
5
Send Your Credentials to Rev.io

Send the following to your Rev.io project contact via a secure channel (encrypted email or secure file sharing — not plain email):

Site URL: na.myconnectwise.net
Company ID: yourcompanyname
Public Key: (from Step 4)
Private Key: (from Step 4)
Frequently Asked Questions
Will this affect my ConnectWise users or data?No. The API key provides read-only access and does not interfere with normal ConnectWise operations.
I already have an API user set up — can I use it?Yes. Generate an additional key pair on an existing API member, or point us to one you already have — just ensure it has the read access described in Step 3.
How do I revoke access after migration?Go to System → Members, open the API Member, click the API Keys tab, and delete the key pair. Access is immediately revoked.
What ConnectWise version is required?ConnectWise Manage version 2019.1 or later. Check under System → My Company → ConnectWise.
